6 Trustworthiness — Reliability — Quality — Clarity

A METR user needs rule information to be unambiguous.
SP:
SP-46 Peer roles need the ability to coordinat…
SP-48 Disseminators need to be able to share c…
 1.06 The data provided by METR will be used by a variety of transport user systems, each with their own applications. The data provided by METR must be sufficiently unambiguous to allow for its use by different users for the varied expected purposes. Transport user systems must be able to interpret the rule information received to the context in which the system is operating. If a road section has a different speed limit for different automobiles and lorries, METR must ensure that the data provided is sufficiently unambiguous to prevent misinterpretation. For example, METR could provide both speed limits or it could provide one while making it clear that the rule does not apply to the other classification of vehicles. A regulation might be defined such that it is in effect between midnight (0:00) and 06:00 when snow is present. For an ADS to comply with this regulation, there must be an agreement on 1) the meaning of midnight (e.g., local time or UTC), identifying which entity (e.g., METR disseminator or ADS) is responsible for determining when the snow present condition is met, 3) whether the regulation applies to the ADS (e.g., does the regulation apply to police vehicles when responding to an emergency). If a user is notified of a static rule for a lower speed limit "when workers are present", it should be aware of whether it should expect to be electronically notified of workers being present and, if so, whether it can legally rely upon that notification.

 1.08 As a part of the network, transport user systems need to be designed to provide safe operations even if the other components of the METR network become unavailable for some period. METR must plan for and adapt to, resist or quickly recover from all failures related to safety critical information. For example, if a traffic signal stops transmitting SPaT messages, the transport user system needs to be able to recognize that the data is missing and take appropriate action. 9 Trustworthiness — Reliability — Timeliness — Static rules

A METR user needs to be able to obtain updated rule sets from the METR distribution system prior to the expiration of the current rule sets.
SP:
SP-94 Each individual rule should have its own…
 1.09 Once rules have been downloaded, it needs to be assured that the rules will not change for an agreed upon period (unless there are push notices); a user system needs to know how often it is required to check for any updates. There are many rules that have been defined by regulators. While many of these rules are relatively static, some can change in an instant. User systems need an accurate representation of rules at every point in time (e.g., on the scale of 0.1 second resolution) to make appropriate dynamic driving task (DDT) decisions. Since it is unreasonable to download all defined rules 10 times a second, user systems will need to store rules for some period. As a result, the user systems will need a way to determine when the previously downloaded information can no longer be assumed to be valid. A user returns to his vehicle after a long hiatus. The vehicle system needs to determine if its previously downloaded rule set is still valid prior to engaging its ADS or presenting information through its driver support system. A parking restriction might be planned to go into effect two weeks from now and then be active from 06:00 to 17:00 Monday through Friday.

12 Trustworthiness — Reliability — Transparency

A METR user needs rule information to be transparent.

 1.12 Transport user systems might make safety-critical operational decisions based on the rule information received. If a collision or other liability situation occurs, forensics will likely need to be performed to ensure what information was available at what point in time. The system design needs to ensure that the rule information received by each user system is sufficiently captured to reliably recreate a meaningful picture of conditions immediately prior to, during, and immediately after the incident. A collision occurs on a road section that has a variable speed limit where the vehicle is believed to have been exceeding the speed limit. To determine liability, investigators will need to determine 1) the speed limit in effect at the time (e.g., as posted on physical infrastructure), 2) the speed limit that the vehicle system was aware of (e.g., as received via METR), and 3) the speed of the vehicle.

 2.04 Preferences should include the costs involved with each communications technology. Transport user systems typically support multiple radios for wireless communication radios. An ad hoc rule might be a simple electronic notice that ad hoc rules are in effect in a defined geofenced area without providing any details (i.e., notice that non-electronic rules are in force). Responders and field crews often impose ad hoc rules.
